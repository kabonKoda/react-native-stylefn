Build react-native-stylefn — A Tailwind-Inspired Auto-Injecting Style Utility for React Native
What You're Building
A React Native styling library that brings Tailwind's constraint-based design token system directly into the style prop of every component in the entire app — automatically, with zero per-component setup.
Developers write style functions instead of style objects. Those functions receive rich contextual tokens — theme, dark mode, breakpoints, orientation, platform, and more — and return a standard RN style object. The library intercepts React's render pipeline to resolve these functions transparently before they hit the component.

Core API — What the Developer Writes
No imports per component. No hooks. No wrappers. No classNames. Just write a function anywhere a style prop is accepted:
jsx<View style={({ dark, colors, breakpoint, orientation }) => ({
  backgroundColor: dark ? colors.background.dark : colors.background.light,
  padding: breakpoint === 'lg' ? 24 : 12,
  flexDirection: orientation === 'landscape' ? 'row' : 'column',
})} />

<Text style={({ theme, fontScale }) => ({
  fontSize: theme.fontSize.base * fontScale,
  color: theme.colors.text,
})} />

<TextInput style={({ dark, colors, insets }) => ({
  backgroundColor: colors.input,
  paddingBottom: insets.bottom + 8,
})} />

// Array styles with mixed functions and objects also work
<View style={[
  { flex: 1 },
  ({ dark }) => ({ backgroundColor: dark ? '#000' : '#fff' }),
  ({ breakpoint }) => breakpoint === 'sm' && { padding: 8 },
]} />
This works on every component in the entire app — RN built-ins, third-party library components, and custom components — as long as they accept a style prop. No enumerated list of supported components. No maintenance. It's universal.

How It Works — Patch React.createElement
The entire system is powered by a single patch applied once at app boot. Instead of targeting specific components, the library intercepts React's own rendering pipeline:
ts// patch.ts
import React from 'react';
import { getTokenStore } from './store';

const resolveStyleProp = (value: unknown) => {
  if (typeof value === 'function') {
    return value(getTokenStore());
  }
  if (Array.isArray(value) && value.some(s => typeof s === 'function')) {
    return value.map(s => typeof s === 'function' ? s(getTokenStore()) : s);
  }
  return value;
};

const originalCreateElement = React.createElement;

React.createElement = (type, props, ...children) => {
  if (props) {
    let patched = false;
    const newProps: Record<string, unknown> = {};

    for (const key in props) {
      // catch style, contentContainerStyle, indicatorStyle, containerStyle, etc.
      if (key.endsWith('Style') || key.endsWith('style')) {
        const resolved = resolveStyleProp(props[key]);
        if (resolved !== props[key]) patched = true;
        newProps[key] = resolved;
      }
    }

    if (patched) {
      props = { ...props, ...newProps };
    }
  }

  return originalCreateElement(type, props, ...children);
};
This intercepts every JSX element rendered in the entire tree. It doesn't care what the component is — only whether props.style is a function. Patch it once inside StyleProvider at the root and it's universal across the whole app forever.

Singleton Token Store
Because the patch runs outside React's render tree, tokens must live in a framework-agnostic singleton that is always synchronously readable:
ts// store.ts
import type { TokenStore } from './types';
import { defaultTokens } from './tokens';

let _store: TokenStore = defaultTokens;

export const getTokenStore = (): TokenStore => _store;
export const setTokenStore = (tokens: TokenStore): void => { _store = tokens; };
StyleProvider subscribes to all device and system events and calls setTokenStore() whenever anything changes — keeping the singleton always in sync with the current device state:
tsx// provider.tsx
export const StyleProvider = ({ children, config }) => {
  const colorScheme = useColorScheme();
  const dimensions = useDimensions();
  const insets = useSafeAreaInsets();
  const reducedMotion = useReducedMotion();
  const fontScale = useFontScale();

  // Apply the patch once on mount
  useEffect(() => { applyPatch(); }, []);

  // Keep singleton in sync
  useEffect(() => {
    setTokenStore(resolveTokens({ colorScheme, dimensions, insets, reducedMotion, fontScale, config }));
  }, [colorScheme, dimensions, insets, reducedMotion, fontScale]);

  return <>{children}</>;
};

Additional Implementation Requirements

TypeScript throughout — full generic types and autocomplete on the TokenStore shape inside every style function
Array style support — mixed arrays of plain objects and functions all resolve correctly
Falsy value handling — false, null, undefined in style arrays are safely ignored (standard RN behavior preserved)
Zero performance overhead when unused — the patch short-circuits immediately if props.style is not a function or array containing functions
useTheme() hook — for accessing tokens inside component logic, event handlers, or animations where a style prop isn't available
useDark() hook — returns { dark, setDark } for manual dark mode control when darkMode: 'manual' is set in config
CSS variable scoping — :root maps to light mode, .dark maps to dark mode, any other selector is ignored with a dev warning
Config hot reload in dev — watch rn-stylefn.config.js for changes and re-resolve tokens without restarting the bundler
Babel plugin (optional, additive) — resolves static style functions at compile time as a pure perf optimization; the runtime patch works fully without it
